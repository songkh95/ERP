이제 A/S접수 페이지를 만들고 싶어
이 많은 내용을 한번에 하면 수정하기가 어려우니까 차근차근 순서대로 하나씩 알려줘 


A/S 접수 페이지는 내가 현재 거래처에 방문하고 왔을 때 기록을 남기는 페이지야

1. 어디에 방문 했는지
2. 언제 방문 했는지
3. 어떤 점검, A/S, 소모품 배송 등 무엇을 했는지 
4. 최근에 언제 방문했고 어떤 내용이였는지 (3번의 내용)
5. 현재 거래처에 두고온 복합기 소모품은 어떤건지 
6. 비고(메모)


[기능]
1. 현재 client에 등록되어 있는 거래처만 점검, A/S 페이지에 나와야 한다.
2. 오늘 점검 내용을 등록하는 화면은 상단에 위치한다.
2-1. 점검 등록 내용: 
- 거래처명(어떤 거래처인지 바로 찾을 수 있도록)
- 모델명(등록된 거래처가 여러대의 복합기를 사용할 수 있기 때문에 client에서 고객에 등록된 복합기가 나오고 선택할 수 있도록 한다.)
- 날짜(점검한 날짜)
- 방문 내용(점검과 a/s 내용을 기입한다.)
- 배송 내용 (어떤 소모품을 배송했는지 토너 혹은 드럼)
- 소모품 여유분(현재 거래처에 어떤 소모품이 남아있는지) 
3. 현재 등록된 거래처 전부가 나타나는데 제일 상단 타이틀은 아래와 같다.
- 거래처명/모델명/날짜/소모품여유분/방문내용/배송내용
- 거래처명을 클리하면 오름차순, 내림차순 변경가능하도록
- 방문내용과 배송내용은 내용이 많이 때문에 내용일 길면 텍스트의 뒷부분은 ...으로 생략
- 점검, A/S 등록된 내용을 클릭했을 때 저장된 모든 내용이 아래로 펼처지도록 한다


******************************************************
데이터베이스

id (int8, PK)
created_at (timestamp, 기본값)
visit_date (date) : 방문 날짜 (필수)
client_id (int8, FK -> clients.id) : 어떤 고객? (필수)
asset_id (int8, FK -> assets.id) : 어떤 기계? (필수)
visit_detail (text) : 방문/점검 내용
delivery_detail (text) : 배송 내용 (토너, 드럼 등)
spare_parts (text) : 소모품 여유분 (현장 재고)
note (text) : 비고 (Nullable)
